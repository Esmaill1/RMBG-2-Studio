# Plan: Make RMBG-2-Studio GPU Server an "Absolute Monster"

## User Decisions (Confirmed)
- **FP8**: OFF by default, require explicit `FP8_INFERENCE=1` (quality risk)
- **Resolution**: Keep 1024×1024, no lower resolution mode (quality priority)
- **TensorRT**: Skip for now, Phase 3 later
- **Output**: PNG with compress_level=1 (NOTE: PNG compression level does NOT affect image quality — PNG is always lossless. compress_level only affects file size vs save time)

## Root Cause Analysis: Why 4 sec/image with only 4GB/139GB used

GPU inference takes ~100ms. The 4 seconds is the **full pipeline**:

| Step | Time | Where | GPU Active? |
|---|---|---|---|
| PIL Image.open() + convert('RGB') | ~0.5-1s | CPU | NO |
| torchvision Resize(1024×1024) + Normalize | ~0.5-1s | CPU | NO |
| GPU model forward pass | ~0.1s | GPU | YES (2.5%!) |
| PIL mask.resize() back to original + putalpha | ~0.3-0.5s | CPU | NO |
| PNG save (lossless compression) | ~1-2s | CPU | NO |
| HTTP request/response round-trip | ~0.3-0.5s | Network | NO |

**GPU is idle ~97.5% of the time.** Frontend sends 1 image per request (app.js line 43-71).

## Implementation Plan (Phase 1 Only — Quick Wins)

### 1. Frontend Batching — THE BIGGEST WIN (8-10x for multi-image)

**File**: `app/static/app.js`

**Current** (line 43-71): Sequential loop, 1 image per request:
```js
for (let i = 0; i < totalFiles; i++) {
    const formData = new FormData();
    formData.append('images', file);  // ONE image per request
    await fetch('/api/remove-bg', ...);
}
```

**Change**: Send ALL images in ONE request:
```js
const formData = new FormData();
for (let i = 0; i < totalFiles; i++) {
    if (files[i].type.startsWith('image/')) {
        formData.append('images', files[i]);
    }
}
const response = await fetch('/api/remove-bg', { method: 'POST', body: formData });
```

- Update progress bar to show "Processing X images in batch..." during the single request
- Update result cards to display all results from the batch response
- Impact: 10 images goes from 40s → ~2-3s

### 2. Increase MAX_CONTENT_LENGTH from 50MB to 200MB

**File**: `app/flask_app_gpu.py` (line 33)

32 images × ~6MB average = ~192MB total upload. Current 50MB limit truncates large batches.

**Change**: `app.config['MAX_CONTENT_LENGTH'] = 200 * 1024 * 1024`

### 3. PNG Fast Save — compress_level=1 (no quality loss!)

**File**: `app/flask_app_gpu.py` (line 306)

**Current**: `processed.save(output_path, 'PNG')` — default compress_level=6, slow (~1-2s per image)

**Change**: `processed.save(output_path, 'PNG', compress_level=1)` — ~0.3-0.5s per image

**IMPORTANT**: PNG compression level does NOT affect image quality. PNG is always lossless regardless of compress_level. compress_level=1 means the encoder does less work to find the smallest byte representation — same pixel-perfect image, just a ~20-30% larger file. The quality is identical.

### 4. GPU-Accelerated Preprocessing with kornia

**File**: `app/flask_app_gpu.py` (lines 191-195, 206-219, 225-256)

**Current**: torchvision CPU transforms:
```python
transform_image = transforms.Compose([
    transforms.Resize((1024, 1024)),
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]),
])
```

**Change**: Replace with kornia GPU transforms. kornia is already in requirements-gpu.txt.

New approach:
```python
import kornia

MEAN = torch.tensor([0.485, 0.456, 0.406], device=device).view(1, 3, 1, 1)
STD = torch.tensor([0.229, 0.224, 0.225], device=device).view(1, 3, 1, 1)

def preprocess_gpu(image):
    """GPU-accelerated preprocessing using kornia"""
    raw = kornia.image_to_tensor(np.array(image)).unsqueeze(0).float().to(device, non_blocking=True)
    resized = kornia.geometry.transform.resize(raw, (1024, 1024), interpolation='bilinear')
    normalized = (resized - MEAN) / STD
    return normalized

def preprocess_gpu_batch(images):
    """GPU-accelerated batch preprocessing"""
    raw_tensors = []
    for img in images:
        raw = kornia.image_to_tensor(np.array(img)).unsqueeze(0).float()
        raw_tensors.append(raw)
    batch_raw = torch.cat(raw_tensors, dim=0).to(device, non_blocking=True)
    resized = kornia.geometry.transform.resize(batch_raw, (1024, 1024), interpolation='bilinear')
    normalized = (resized - MEAN) / STD
    return normalized
```

Impact: ~0.5-1s of CPU transform time per image → ~5ms on GPU. Batch of 32 images: all resized/normalized in one GPU pass (~50ms).

### 5. Pipeline Parallelism — Async File Saving

**File**: `app/flask_app_gpu.py`

**Current**: Sequential — GPU inference → then save all files → then return response.

**Change**: Use `concurrent.futures.ThreadPoolExecutor` to overlap file saving with response preparation:

```python
from concurrent.futures import ThreadPoolExecutor

save_executor = ThreadPoolExecutor(max_workers=4)

def save_result(processed, output_path):
    processed.save(output_path, 'PNG', compress_level=1)
    return output_path

# In api_remove_bg:
save_futures = []
for i, processed in enumerate(processed_images):
    ...
    future = save_executor.submit(save_result, processed, output_path)
    save_futures.append(future)

# Build response while saves happen in background
results = [...]  # Build result dicts while saves are running

# Wait for all saves to complete before returning
for future in save_futures:
    future.result()
```

Impact: While CPU saves batch N, next request can start loading images. Saves ~1-2s of blocking time.

### 6. Enable torch.compile on Datacenter GPUs (24+GB VRAM)

**File**: `app/flask_app_gpu.py` (line 113)

**Current**: `recommended_compile = '0'` (always OFF)

**Change**: Enable on datacenter GPUs (24+GB) where the server runs continuously:
```python
recommended_compile = '1' if gpu_mem_total >= 24 else '0'
```

For H200 (139.8GB): ON by default. Smaller GPUs still get OFF.
The 5-15 min startup delay is a one-time cost for a continuously running server.
Speedup: 1.5-2x per-image inference.

### 7. Scale MAX_BATCH_SIZE for 139.8GB H200

**File**: `app/flask_app_gpu.py` (line 105)

**Current**: MAX_BATCH_SIZE=32 (capped even for 139.8GB)

**Change**: Scale aggressively for large VRAM:
```python
recommended_batch = 4 if gpu_mem_total < 6 else 8 if gpu_mem_total < 12 else \
                    16 if gpu_mem_total < 24 else 32 if gpu_mem_total < 48 else \
                    64 if gpu_mem_total < 96 else 128
```

H200 (139.8GB): MAX_BATCH_SIZE=128. Model (176MB) + 128 images (768MB) ≈ 1GB total on 139.8GB.
